# E-012 — saveArtifactVersion: agent_name pasado como UUID

## Evidencia de logs

```
[p2-assembler] CCM saveArtifactVersion falló (no crítico):
  saveArtifactVersion insert failed: 
  invalid input syntax for type uuid: "ensamblador_doc_p2"
  
[p5-assembler] CCM saveArtifactVersion falló (no crítico):
  saveArtifactVersion insert failed: 
  invalid input syntax for type uuid: "ensamblador_doc_p5"
```

## Análisis de causa raíz

La función `saveArtifactVersion` recibe `generatedBy: string` y lo inserta en la columna `generated_by UUID` de `artifact_versions`. El llamador pasa el nombre del agente (`"ensamblador_doc_p2"`) en lugar del `userId` UUID del usuario que ejecuta el pipeline.

El campo `generated_by` debería ser el UUID del usuario (disponible en el job context), no el nombre del agente. El agente debería almacenarse en una columna separada (`model` o un nuevo campo `agent_name`).

## Impacto

El sistema CCM (Gestión de Versiones de Artefactos) está completamente roto para P2–P8. Ningún artefacto generado se versiona. El expediente de certificación está vacío.
