# E-011 — Inyección de Project Soul y Project Brief en enrichedContext

## Contenido capturado

```typescript
// document.handlers.ts:208-239
try {
  const soul = await getProjectSoul(projectId);
  if (soul) {
    enrichedContext._projectSoul = soul;
  }
} catch (e) {
  logger.warn('[pipeline] getProjectSoul failed, continuing without soul', e);
}

try {
  const brief = await getProjectBrief(projectId);
  if (brief) {
    enrichedContext._projectBrief = brief;
    enrichedContext._frozen = {
      nombreOficialCurso: brief.nombreOficialCurso,
      dominioTecnico: brief.dominioTecnico,
      resultadoCentral: brief.resultadoCentral,
      audienciaPrimaria: brief.audienciaPrimaria,
    };
  }
} catch (e) {
  logger.warn('[pipeline] getProjectBrief failed, continuing without brief', e);
}
```

## Análisis

- Ambas inyecciones están en try-catch separados — fallo de una no afecta la otra.
- Los campos `_frozen.*` son las anclas semánticas inmutables que los prompts pueden referenciar.
- Los logs en vivo confirman que esta inyección funciona correctamente en producción.
- En tests, los mocks de `supabase.service.ts` no exponen `getProjectSoul` ni `getProjectBrief`, causando fallo silencioso en el contexto de pruebas (H-005).
