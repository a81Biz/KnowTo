# H-011 — TAVILY_API_KEY ausente en `.dev.vars`: OSINT falla silenciosamente en dev nativo

## Descripción

El `WebSearchService` (`enrichContextWithOSINT`) enriquece el contexto antes de los pipelines LLM con información actualizada del web. Sin `TAVILY_API_KEY`, el enriquecimiento falla silenciosamente (catch en línea 407 de `document.handlers.ts`).

En Docker, `TAVILY_API_KEY=tvly-dev-...` está configurado en `docker-compose.yml`.  
En desarrollo nativo (tests, `npm run dev`), la variable no está en `src/backend/.dev.vars` → OSINT no funciona.

Evidencia en stderr de tests:
```
[WebSearchService] TAVILY_API_KEY no configurada. Las búsquedas fallarán.
enrichContextWithOSINT falló — continuando sin OSINT: Cannot read properties of null (reading 'from')
```

## Impacto

Bajo en producción (Docker siempre tiene la key). Medio en desarrollo nativo: los developers que ejecuten `npm run dev` directamente o los tests no recibirán el enriquecimiento OSINT y los documentos generados no tendrán contexto web actualizado.

## Recomendación

1. Añadir `TAVILY_API_KEY` a `.dev.vars.example` con el valor dev
2. Documentar en README.md que `TAVILY_API_KEY` es necesaria para el enriquecimiento OSINT
3. Añadir mock de `WebSearchService` en el setup de tests para evitar el error de `null` en `osint.helper.ts`
