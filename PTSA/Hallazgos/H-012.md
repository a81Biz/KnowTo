# H-012 — `saveArtifactVersion` falla con UUID inválido en P2–P8: se pasa nombre de agente como `generated_by`

## Descripción

Los logs del backend muestran errores repetidos en todos los assemblers de P2, P5, P6, P7 y P8:

```
[p2-assembler] CCM saveArtifactVersion falló (no crítico):
  saveArtifactVersion insert failed: invalid input syntax for type uuid: "ensamblador_doc_p2"
```

El campo `generated_by` en la tabla `artifact_versions` es de tipo `uuid`, pero los assemblers están pasando el nombre del agente (`"ensamblador_doc_p2"`, `"ensamblador_doc_p5"`, etc.) como si fuera un UUID.

## Evidencia

Logs en vivo (docker logs knowto-backend):
```
[p2-assembler] CCM saveArtifactVersion falló (no crítico): invalid input syntax for type uuid: "ensamblador_doc_p2"
[p5-assembler] CCM saveArtifactVersion falló (no crítico): invalid input syntax for type uuid: "ensamblador_doc_p5"
[p6-assembler] CCM saveArtifactVersion falló (no crítico): invalid input syntax for type uuid: "ensamblador_doc_p6"
[p7-assembler] CCM saveArtifactVersion falló (no crítico): invalid input syntax for type uuid: "ensamblador_doc_p7"
[p8-assembler] CCM saveArtifactVersion falló (no crítico): invalid input syntax for type uuid: "ensamblador_doc_p8"
```

Schema `artifact_versions.generated_by`: `uuid`, nullable.

## Análisis de causa raíz

Los assemblers P2–P8 (y posiblemente P1, P3, P4) están llamando a `saveArtifactVersion` pasando `agentName` (string) en el campo `generated_by` en lugar del UUID del usuario (`userId`) o `null`. El campo `generated_by` debería ser el UUID del usuario que disparó la generación, no el nombre del agente.

## Impacto

**Alto.** El sistema de versionado de artefactos (`artifact_versions`) está completamente roto para P2–P8. Ningún artefacto se está guardando en `artifact_versions`, lo que significa:
- La trazabilidad de qué modelo/prompt generó qué versión del documento no se está almacenando
- El campo `active_artifact_id` en `fase4_productos` probablemente apunta a registros que no existen
- El CCM (Content Change Management) no está funcionando

El error se marca como "no crítico" por lo que el documento se guarda en `fase4_productos` de todas formas, pero el sistema de versionado falla silenciosamente.

## Recomendación

En cada assembler (p2-document.assembler.ts, p5–p8-document.assembler.ts), corregir la llamada a `saveArtifactVersion`:
- `generated_by` debe ser `event.body?.userId ?? null` (UUID del usuario)
- No pasar el nombre del agente como `generated_by`

```typescript
// INCORRECTO:
await saveArtifactVersion({ ..., generated_by: agentName });

// CORRECTO:
await saveArtifactVersion({ ..., generated_by: event.body?.userId ?? null });
```
