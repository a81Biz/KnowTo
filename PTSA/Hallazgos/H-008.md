# H-008 — P1 (Instrumentos de Evaluación) rechazado por el EC0366RulesEngine

## Descripción

El documento de Instrumentos de Evaluación (F4-P1) fue generado y almacenado con estado `rejected` en `fase4_productos`. Las violaciones detectadas por el `EC0366RulesEngine` son:

1. **"Reactivos contienen palabras subjetivas prohibidas"** — el documento usa vocabulario no medible (ej: "adecuado", "correcto", "bien", "efectivo") en los reactivos de evaluación, lo cual viola EC0366 que exige verbos de acción medible
2. **"Combina dos tipos de instrumento en la misma unidad"** — la Unidad 1 tiene `Instrumento: Guía de Observación / Lista de Cotejo`, mezcla dos tipos cuando EC0366 requiere un solo tipo por unidad

## Evidencia

Query psql `fase4_productos`:
```json
validacion_estado: "rejected"
validacion_errores: {
  "errors": [
    "Reactivos contienen palabras subjetivas prohibidas (adecuado/correcto/bien/efectivo/entendimiento/comprensión)",
    "Combina dos tipos de instrumento en la misma unidad"
  ],
  "passed": false
}
```

Documento P1 preview:
```
## Unidad 1: Introducción al Proceso de Acreditación por Acuerdo 286
- **Instrumento:** Guía de Observación / Lista de Cotejo
```

## Análisis de causa raíz

El modelo LLM generó el instrumento con dos tipos mezclados (`Guía de Observación / Lista de Cotejo`) en lugar de elegir uno. El `EC0366RulesEngine` detectó correctamente la violación y rechazó el documento.

El mecanismo de reintento `p1-retry.helper.ts` existe para este caso, pero en el run auditado no fue suficiente para producir un P1 aprobado.

## Impacto en certificación

**ALTO.** Los Instrumentos de Evaluación (P1) son uno de los entregables más críticos de EC0366 E1220. Un P1 en estado `rejected` significa que el documento actual NO puede usarse para la solicitud de certificación CONOCER sin intervención humana. El cliente debe revisar y corregir manualmente.

## Recomendación

1. Verificar si `p1-retry.helper.ts` está correctamente integrado en el pipeline como retry automático tras la primera falla
2. Reforzar el prompt de F4_P1_GENERATE_DOCUMENT con instrucción explícita: "NUNCA uses '/' entre dos tipos de instrumento. Elige UN SOLO tipo por unidad."
3. Añadir validación en el assembler que detecte el patrón `Tipo / Tipo` antes de llamar al RulesEngine y rechace el output del LLM antes de guardarlo
