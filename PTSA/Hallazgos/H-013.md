# H-013 — Columna `artifact_versions.status` no existe en BD pero el código la consulta

## Descripción

Los logs del backend muestran un error en la ruta de certificación:

```
[certification-route] error: Error: artifact_versions query failed:
  column artifact_versions.status does not exist
```

El código en la ruta de certificación consulta `artifact_versions.status`, pero esa columna no fue creada en la migración 046 (`046_artifact_versions.sql`). El schema real de la tabla confirma que las columnas existentes son: `id`, `project_id`, `product_code`, `version`, `artifact`, `documento_md`, `prompt_template_id`, `prompt_template_version`, `prompt_hash`, `model`, `generated_by`, `cert_score`, `correction_log`, `derived_from_artifact_id`, `created_at`, `is_active`.

No hay columna `status`.

## Evidencia

Logs en vivo:
```
[certification-route] error: Error: artifact_versions query failed:
  column artifact_versions.status does not exist
    at async <anonymous> (/app/src/core/middleware/error.middleware.ts:6:5)
```

Schema verificado via psql (`information_schema.columns`): columna `status` ausente en `artifact_versions`.

## Análisis de causa raíz

El código de la ruta de certificación fue escrito asumiendo que `artifact_versions` tendría una columna `status`, pero la migración 046 no la creó. Hay un desacoplamiento entre el esquema de BD planificado y el esquema de BD real.

Posibles escenarios:
1. La columna `status` estaba en un borrador de la migración que se simplificó usando `is_active` en su lugar
2. La ruta de certificación fue escrita anticipando una futura migración que aún no se aplicó

## Impacto

**Alto.** La ruta de certificación (`certification-route`) falla con un error 500 siempre que intenta consultar `artifact_versions`. Esto significa que cualquier endpoint que exponga scores de certificación o historial de versiones de artefactos está completamente roto. Los clientes que intenten acceder al expediente de certificación recibirán un error.

## Recomendación

**Opción A (preferida):** Corregir la query para usar `is_active` en lugar de `status`:
```sql
-- INCORRECTO: WHERE status = 'active'
-- CORRECTO:   WHERE is_active = true
```

**Opción B:** Añadir la migración faltante `048_add_artifact_versions_status.sql` con la columna `status`.

Verificar en el código fuente de `certification-route` qué valores de `status` se esperan y mapearlos a `is_active` si la semántica lo permite.
