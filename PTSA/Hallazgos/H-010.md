# H-010 — Objetivo subjetivo en Temario Base se propaga a P4 y P1

## Descripción

El Temario Base (ancla canónica) contiene un objetivo de unidad con expresión subjetiva prohibida por EC0366:

```
"objetivo_bloom": "Evalúa el nivel actual de conocimiento sobre los requisitos
                   y proceso de acreditación por Acuerdo 286."
```

La expresión **"nivel actual de conocimiento"** es vaga/subjetiva. No es un indicador observable ni medible:
- ❌ "nivel actual de conocimiento" — ¿cómo se mide? ¿cuál es el nivel esperado?
- ✅ Alternativa correcta: "identifica los requisitos para solicitar acreditación por Acuerdo 286 listando al menos 5 documentos requeridos"

Este objetivo fue propagado a P4 (Manual del Participante): "Evaluarás el **nivel actual de conocimiento** sobre los requisitos..." → detectado como `aprobado_con_errores`.

## Evidencia

- Temario Base BD: `"objetivo_bloom": "Evalúa el nivel actual de conocimiento sobre los requisitos y proceso de acreditación por Acuerdo 286."`
- P4 objetivo Capítulo 1: "Evaluarás el nivel actual de conocimiento sobre los requisitos y proceso de acreditación por Acuerdo 286."
- P4 `validacion_estado`: `aprobado_con_errores`
- `validacion_errores`: `{"errors": ["Palabras subjetivas prohibidas detectadas en el documento final"]}`

## Análisis de causa raíz

El `EC0366RulesEngine` detecta "conocimiento" como palabra prohibida (`PROHIBITED_VERBS` incluye `'entendimiento'`, `'comprensión'`). La raíz del problema está en el **TEMARIO_BASE.md template** o en la fase del temario que no aplicó el filtro de verbos prohibidos a los objetivos de unidad antes de guardar.

El `temario.phase.ts` no parece pasar los objetivos generados por el `EC0366RulesEngine` antes de guardar en BD.

## Impacto

Medio-Alto. El Temario Base es el documento de mayor criticidad — es la ancla de todos los productos. Si el temario tiene objetivos con vocabulario prohibido, todos los productos que lean del temario heredarán el problema.

## Recomendación

1. Añadir validación de verbos Bloom en `temario.phase.ts` antes de guardar en `temario_base`:
   - Rechazar o logear como advertencia si los objetivos contienen palabras subjetivas prohibidas
2. Añadir reglas explícitas en `TEMARIO_BASE.md` que instruyan al modelo:
   - "Los objetivos de cada unidad DEBEN usar verbos de taxonomía de Bloom medibles y observables. PROHIBIDO: 'conocimiento', 'comprensión', 'entendimiento', 'adecuado'."
3. Considerar que el `EC0366RulesEngine` valide también el `temario_base` antes de marcarlo como `confirmado_por_usuario=true`.
