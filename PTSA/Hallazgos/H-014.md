# H-014 — Vulnerabilidades npm: 1 crítica, 3 altas, 5 moderadas en dependencias de build

## Descripción

`npm audit` en el backend reporta vulnerabilidades activas:
- **1 crítica** (esbuild ≤0.28.0: via tsx + wrangler)
- **3 altas** (esbuild: RCE via NPM_CONFIG_REGISTRY, GHSA-gv7w-rqvm-qjhr)
- **5 moderadas** (miniflare vía undici/ws)

Las vulnerabilidades son todas en dependencias de **build/dev** (`tsx`, `wrangler`, `miniflare`), no en dependencias de runtime de producción. Sin embargo, algunas afectan el servidor de desarrollo (`npm run dev`).

## Evidencia

```
9 vulnerabilities (5 moderate, 3 high, 1 critical)
- esbuild: Missing binary integrity verification → RCE via NPM_CONFIG_REGISTRY (GHSA-gv7w-rqvm-qjhr)
- esbuild: Any website can send requests to dev server (GHSA-67mh-4wv8-2f99)
- esbuild: Arbitrary file read on Windows dev server (GHSA-g7r4-m6w7-qqqr)
- miniflare: vía undici, ws
```

Fix disponible: `wrangler` upgrade a v4.86.0 (semver major).

## Impacto

**Medio en dev local, bajo en producción.** Las vulnerabilidades de esbuild afectan al servidor de desarrollo (`npm run dev`) que corre con `tsx`. En producción (Cloudflare Workers), el runtime es `workerd` y no usa esbuild directamente. Sin embargo, la vulnerabilidad RCE de esbuild via `NPM_CONFIG_REGISTRY` es preocupante para desarrolladores que ejecuten `npm install` en entornos no confiables.

## Recomendación

1. Actualizar `wrangler` a ≥4.86.0 en `package.json` (semver major — evaluar breaking changes)
2. Actualizar `tsx` a versión que use esbuild ≥0.28.1
3. Ejecutar `npm audit fix` después de los upgrades para verificar resolución
