# H-003 — README menciona modelo Ollama obsoleto (`llama3.2:3b` → `qwen2.5:14b`)

## Descripción

El README.md y PROYECTO.md documentan el modelo de Ollama como `llama3.2:3b`. La realidad operativa es:
- `docker-compose.yml`: `OLLAMA_MODEL: "qwen2.5:14b"` 
- `docker-compose.yml` entrypoint Ollama: `ollama pull qwen2.5:14b && ollama pull gemma2:27b`
- `TEMARIO_BASE.md` template: `model: "qwen2.5:14b"` (explícito en el frontmatter del pipeline)
- `docker-compose.yml` ya no incluye `llama3.2:3b` en ninguna parte

Esta es una actualización significativa (modelos mucho más grandes: 3B → 14B + 27B) que cambia:
1. Los requisitos de hardware (RAM mínima, tiempo de respuesta)
2. La calidad esperada de los outputs
3. El tiempo de pull inicial del modelo

## Evidencia

- README.md línea: "IA — desarrollo | Ollama local — `llama3.2:3b`"
- docker-compose.yml línea 91: `OLLAMA_MODEL: "qwen2.5:14b"`
- docker-compose.yml líneas 380-385: `ollama pull qwen2.5:14b && ollama pull gemma2:27b`
- TEMARIO_BASE.md frontmatter: `model: "qwen2.5:14b"`

## Impacto

Un desarrollador nuevo siguiendo el README intentará usar `llama3.2:3b` y tendrá outputs inconsistentes. Los requisitos de hardware documentados (implícitamente para un modelo 3B) son insuficientes para los modelos 14B+27B.

## Recomendación

Actualizar README.md tabla de stack técnico con el modelo real `qwen2.5:14b` + `gemma2:27b`. Actualizar la sección de tiempos esperados y requisitos de hardware.
