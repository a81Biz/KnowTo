# H-006 — `graphify-out/` embebido dentro de `src/backend/src/`

## Descripción

Existe un directorio `src/backend/src/graphify-out/cache/` que parece ser la salida del tool de knowledge graph (`graphify`) colocada dentro del directorio de código fuente del backend.

El directorio de graphify debería estar en la raíz del repositorio (`graphify-out/` ya existe en raíz) o en un directorio de análisis, no dentro de `src/backend/src/` ya que:
1. Puede ser incluido en el bundle de producción
2. Contamina el árbol de fuentes
3. Puede confundir a herramientas de búsqueda de código

## Evidencia

- `c:\DevOps\Desarrollos\KnowTo\src\backend\src\graphify-out\cache` (existe)
- `c:\DevOps\Desarrollos\KnowTo\graphify-out\` (también existe en raíz, correctamente)

## Impacto

Bajo. No afecta la funcionalidad en producción (a menos que el bundler de Cloudflare Workers incluya archivos de este directorio). Pero agrega ruido al árbol de fuentes.

## Recomendación

Verificar si `graphify-out/` está en `.gitignore` para `src/backend/src/`. Si no, añadirlo. Mover o eliminar el directorio de `src/backend/src/graphify-out/`.

## Revisión — PT-193 (2026-06-18)

**Estado:** CORREGIDA

**Corrección aplicada:**
1. Directorio `src/backend/src/graphify-out/` eliminado del filesystem.
2. `src/backend/tsconfig.json`: añadido `"src/graphify-out"` al array `exclude` como prevención ante futuras ejecuciones de graphify desde dentro de src/.

**Validación:** `ls src/backend/src/` no muestra el directorio. 361/361 tests pasan (ningún archivo .ts importaba desde graphify-out).
