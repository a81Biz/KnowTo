# H-005 — Test mocks desactualizados: `getProjectSoul` y `getFase0Estructurado` ausentes en mocks

## Descripción

Durante la ejecución de la suite de tests, aparecen errores en stderr que indican que los mocks de `SupabaseService` usados en los tests no implementan métodos que sí existen en producción:

```
TypeError: supabase.getProjectSoul is not a function
TypeError: supabase.getFase0Estructurado is not a function
```

Estos errores ocurren en `document.handlers.ts`:
- Línea 209: `supabase.getProjectSoul(body.projectId)` — inyecta el Project Soul (Semantic Anchor Layer)
- Línea 231: `supabase.getFase0Estructurado(body.projectId)` — inyecta el F0 estructurado en contexto F1

Los tests siguen pasando porque ambas llamadas están en bloques `try-catch`, pero:
1. La inyección del **Project Soul** falla silenciosamente en el entorno de test
2. La inyección de **F0 estructurado en F1** falla silenciosamente en el entorno de test
3. Los tests no validan si estas inyecciones ocurrieron correctamente

## Evidencia

- Stderr test run 2026-06-13: `TypeError: supabase.getProjectSoul is not a function at buildEnrichedContext (document.handlers.ts:209)`
- Stderr: `TypeError: supabase.getFase0Estructurado is not a function at buildEnrichedContext (document.handlers.ts:231)`
- `src/backend/src/core/services/supabase.service.ts:902`: `async getProjectSoul(projectId: string)`
- `src/backend/src/dcfl/services/supabase.service.ts:68`: `async getFase0Estructurado(projectId: string)`

## Impacto

**Medio.** Los tests no prueban el comportamiento del sistema cuando el Project Soul y F0 estructurado son inyectados correctamente. Cualquier bug en la lógica de inyección (por ejemplo, qué pasa si el contexto tiene Project Soul pero el template no lo usa correctamente) no sería detectado por los tests actuales. Esto reduce la confianza en la cobertura de pruebas para el Semantic Anchor Layer, que es documentado como crítico en CLAUDE.md.

Adicionalmente hay un tercer error:
- `enrichContextWithOSINT falló: Cannot read properties of null (reading 'from')` — el servicio de web search no está mockeado correctamente

## Recomendación

Actualizar los mocks de `SupabaseService` en los tests para incluir:
1. `getProjectSoul`: puede retornar un string vacío o un project soul de prueba
2. `getFase0Estructurado`: puede retornar un objeto vacío o datos de F0 de prueba
3. Añadir assertions que verifiquen que `ctx._projectSoul` fue inyectado cuando el método retorna datos

## Revisión — 2026-06-15

**Estado:** CORREGIDA ✅

Ambos archivos de test actualizados con los mocks faltantes:

- **wizard.async.e2e.test.ts**: añadidos `getProjectSoul`, `getFase0Estructurado`, `getFaseAnswersDetailed`, `confirmarTemario`, `getTemarioBase`, `getProjectBrief`, `saveProjectBrief`.
- **wizard.e2e.test.ts**: mismo conjunto de mocks añadido.
