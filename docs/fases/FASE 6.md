## PROCESO QUE DEBES SEGUIR

### PASO 1: Clasificar las observaciones corregidas

Toma el reporte de revisión que el usuario llenó. Por cada observación, el usuario debe haber indicado si fue corregida.

Clasifica en:

| Estado | Descripción | Acción en este documento |
|:---|:---|:---|
| **Corregida** | La observación se resolvió | Documentar la acción correctiva |
| **Pendiente** | No se resolvió (por decisión o por imposibilidad) | Documentar por qué y plan de acción |
| **Rechazada** | Se decidió no corregir (por justificación) | Documentar la justificación |

### PASO 2: Documentar cada acción correctiva

Para cada observación corregida, registra:

- **Observación original** (copiada del reporte)
- **Acción correctiva tomada** (qué se cambió exactamente)
- **Fecha de corrección** (cuándo se hizo)
- **Verificación** (cómo se comprobó que la corrección funcionó)
- **Responsable** (quién hizo la corrección)

## INSTRUCCIONES DE CALIDAD

1. **No asumas que todas las observaciones fueron corregidas.** El usuario debe indicar explícitamente el estado de cada una.

2. **Si hay observaciones pendientes o rechazadas,** deben estar justificadas. No se permite "no lo hice porque no quise".

3. **Este documento cierra el ciclo de la Fase 6** ("Revisión del plan en marcha" según Sesión 9-10, pág. 2). Sin este documento, el proyecto no está completo.

4. **Una vez que el usuario confirme este documento,** el curso está oficialmente terminado y listo para implementar con participantes reales.
